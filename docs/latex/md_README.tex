 {\ttfamily pal\+\_\+os} is a portable C library that provides an abstraction layer for core operating system functionality, simplifying cross-\/platform development and enabling reusable code.\hypertarget{md_README_autotoc_md1}{}\doxysection{\texorpdfstring{✨ Features}{✨ Features}}\label{md_README_autotoc_md1}
This module includes abstractions for\+:


\begin{DoxyItemize}
\item 🧵 {\bfseries{Threads}} — Platform-\/independent thread creation and management
\item 🔒 {\bfseries{Mutexes/queues}} — Synchronization and inter-\/task communication (Free\+RTOS-\/style)
\item 📶 {\bfseries{Signals/events}} — Lightweight mechanisms for asynchronous notification
\item ⏱️ {\bfseries{Time management}} — Absolute and relative time, delays, time measurement
\item ⏲️ {\bfseries{Software timers}} — One-\/shot and periodic timers with callbacks
\end{DoxyItemize}\hypertarget{md_README_autotoc_md2}{}\doxysection{\texorpdfstring{🛠️ Goal}{🛠️ Goal}}\label{md_README_autotoc_md2}
The main goal is to provide a consistent, minimal API for abstracting OS features, enabling\+:


\begin{DoxyItemize}
\item seamless portability between embedded systems (e.\+g., Free\+RTOS, ESP-\/\+IDF) and host platforms (Linux)
\item modular and testable code
\item integration in complex projects without heavyweight dependencieP

The library can be built both for host systems and embedded targets (e.\+g., ESP32). 
\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/B\ build\ -\/DTARGET\_PLATFORM=linux}

\end{DoxyCode}
